# 한국사 학습 콘텐츠 작성 가이드

이 문서는 한국사능력검정시험 **심화(1~3급)** 대비 학습 앱의 콘텐츠 데이터 파일을 작성하는 기준이다.

## 먼저 읽을 파일
1. `src/lib/types.ts` — `DayContent`, `StudyCard`, `Quiz` 타입 정의
2. `src/data/curriculum.ts` — 담당 일차(day)의 `title`과 `topics` (반드시 이 브리프를 기준으로 작성)

## 출력 파일 형식

지정받은 경로에 아래 형식의 **유효한 TypeScript 파일**을 작성한다:

```ts
import type { DayContent } from "@/lib/types";

export const days: DayContent[] = [
  {
    day: 10,
    cards: [
      {
        id: "d10-c01",
        title: "광개토대왕",
        keywords: ["영락", "만주 진출", "정복 군주"],
        content: "고구려 제19대 왕으로, 우리나라 최초의 연호인 '영락'을 사용하였다. 거란·후연·백제를 격파하고 만주와 요동 일대로 영토를 크게 확장하였다.",
      },
      {
        id: "d10-c02",
        title: "호우명 그릇",
        keywords: ["경주 호우총", "광개토대왕", "신라-고구려 관계"],
        content: "경주 호우총에서 발견된 청동 그릇으로, 밑바닥에 '을묘년국강상광개토지호태왕(乙卯年國岡上廣開土地好太王)'이라는 글자가 새겨져 있다. 5세기 신라가 고구려의 정치적 영향력 아래 있었음을 보여 주는 유물이다.",
        imageSearch: "Silla Houu bronze bowl Gwanggaeto",
      },
      // ... 20~25장
    ],
    quizzes: [
      {
        id: "d10-q01",
        question: "경주 호우총에서 출토된 '호우명 그릇'에 이름이 새겨진 왕의 업적으로 옳은 것은?",
        options: [
          "평양으로 수도를 옮기고 남진 정책을 추진하였다.",
          "영락이라는 연호를 사용하고 만주 일대로 영토를 확장하였다.",
          "태학을 설립하고 율령을 반포하였다.",
          "진대법을 실시하여 빈민을 구제하였다.",
        ],
        answer: 1,
        explanation: "호우명 그릇의 주인공은 광개토대왕이다. 평양 천도는 장수왕, 태학·율령은 소수림왕, 진대법은 고국천왕의 업적이다.",
      },
      // ... 12~15문항
    ],
  },
  // ... 담당한 나머지 일차
];
```

## 분량
- **카드: 하루 20~25장** — 그날 `topics`의 모든 항목을 빠짐없이 커버하고, 심화 시험 빈출 포인트를 추가
- **퀴즈: 하루 12~15문항** — 4지선다

## 카드 작성 규칙
- `title`: 핵심 키워드 하나 (인물·사건·제도·유물 등). 짧게. 예: "노비안검법", "무령왕릉", "훈요 10조"
- `keywords`: 연관 키워드 2~5개 (연도, 관련 인물, 연관 개념)
- `content`: **2~3문장, 120~200자.** 시험에 출제되는 포인트 중심으로. 정의 → 의의/시험 포인트 순으로
- 카드 순서는 그날 주제의 흐름(시간순/논리순)을 따른다
- 같은 날 안에서 내용 중복 금지

## 퀴즈 작성 규칙
- 문제 유형을 다양하게 섞는다:
  - 자료 제시형: "다음 유물/정책/사료와 관련된 왕(사건)은?" (심화 시험의 주력 유형)
  - 업적/내용 일치형: "~왕의 업적으로 옳은 것은?"
  - 시기/순서형: "다음 사건들을 순서대로 나열한 것은?" / "(가) 시기에 있었던 사실은?"
  - 옳지 않은 것 고르기
- 오답 선지는 **그럴듯하게**: 같은 시대의 다른 인물/제도, 또는 수험생이 자주 헷갈리는 다른 시대의 항목을 사용
- `answer` 인덱스(0~3)는 문항마다 고르게 분산시킨다 (한쪽에 몰리면 안 됨)
- `explanation`: 정답 근거 1문장 + 주요 오답이 왜 아닌지 1문장
- 그날 카드에서 다룬 내용 범위 안에서 출제한다

## 사실 정확성 (가장 중요)
- 교과서·기출 통설만 사용한다. 학계 논쟁이 있거나 불확실한 내용은 쓰지 않는다
- 연도·인명·지명·저서명은 확실한 것만 명시한다. 확신이 없으면 연도를 빼고 세기로 표현한다
- 지어내지 않는다. 확실하지 않은 세부 사항은 아예 다루지 않는다

## 이미지 (`imageSearch`)
- **시각 자료가 시험 포인트인 항목에만** 붙인다: 유물, 문화재, 건축물, 회화, 비석, 지도, 근현대 인물 사진 등
- 값은 Wikimedia Commons에서 검색될 법한 **영문 검색어** (로마자 표기 + 영문 설명 조합)
  - 예: `"Bulguksa temple Gyeongju"`, `"Gilt-bronze Maitreya in Meditation"`, `"Tripitaka Koreana Haeinsa"`, `"Donghak Peasant Revolution Jeon Bongjun"`
- 전체 카드의 15~30% 수준. 개념·제도처럼 시각 자료가 무의미한 항목에는 붙이지 않는다
- `image` 필드는 채우지 않는다 (빌드 단계에서 자동 resolve)
- 퀴즈에도 자료 제시형 문제에 한해 `imageSearch`를 붙일 수 있다

## ID 규칙
- 카드: `d{일차 2자리}-c{순번 2자리}` — 예: day 9의 3번째 카드 → `"d09-c03"`
- 퀴즈: `d{일차 2자리}-q{순번 2자리}` — 예: `"d09-q11"`

## 문법 주의
- 문자열은 큰따옴표(`"`)로 감싸고, 내용 안의 큰따옴표는 `\"`로 이스케이프하거나 작은따옴표/낫표(『』, ')를 사용
- 백틱(`` ` ``) 템플릿 문자열 사용 금지
- 파일은 반드시 문법 오류 없는 TypeScript여야 한다

## 완료 보고
파일 작성 후, 일차별 카드/퀴즈 개수만 한 줄씩 요약해서 보고한다 (내용 전문은 보고하지 말 것).
